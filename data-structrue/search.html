# 顺序搜索

最坏的时间复杂度为O(n)，最好的为O(1），这个是搜索效率最低的搜索方式

  • 核心思想是：遍历数组中的每一个元素，和目标元素相互比较，最终找出目标元素所在的位置
Array.prototype.search = function(target){
    for(let i=0;i<this.length;i++){
        if(target === this[i]){
            return i
        }
    }
    return -1
}

# 二分搜索

最好的时间复杂度为O(1),最差的时间复杂度为O(log(n))

  • 实现思路一：将数组一份为二，每次取中间值和目标值比较
  1. 将数组从小到大排序
  2. 取首尾两个指针，取数组的中间值和目标元素比较，相等直接返回该索引
  3. 如果中间值大于目标元素，就需要在左边数组元素中找，需要移动尾部索引
  4. 如果中间值小于目标元素，就需要在右边数组元素中找，需要移动首部索引
