# JS 闭包

# 闭包概念

  • MDN定义：闭包是指那些能够访问自由变量的函数
  • 自由变量：在函数中使用的，既不是函数的参数也不是函数的局部变量
  • 那么我们可以得出闭包的条件是：闭包 = 函数 + 函数能够访问自由变量
var value = 1
function print(){
    console.log(value)
}
print()

解析：上面的例子中print是一个函数，同时print中也访问的自由变量value,那这不就形成了一个闭包，所以在JavaScript权威指南中就指出，所有的JavaScript函数都属于闭包

  • 这个理解和我们平时的认知有些差别，其实上面的是理论上的闭包，还有实践上的闭包
  • 理论角度：js中所有的函数，在函数创建的时候就已经保存了上层的上下文的变量对象到自己作用域中，访问最外层的全局变量也是，访问全局变量也是在访问自由变量
  • 实践角度：1、即使父作用域执行完成被销毁了，它依然存在（被保存到了外部），2、函数内部中访问了自由变量
  • 我在作用链的那一章也进行了解析，下面再来贴一次
var scope = 'global scope'
function f1(){
    var scope = 'local scope'
    function f2(){
        return scope
    }
    return f2
}
f1()()

1、这段代码执行，先创建全局的上下文栈，这个时候上下文栈为

ECStack = [globalContext]

2、进入全局的上下文的同时，函数f1也被定义，f1的scope会将上层的作用域链到内部[[scope]]

globalContext = {
    VO:{
        scope,
        f1
    },
    scope:[globalContext.VO],
    this:globalContext.VO
}

f1.[[scope]] = [globalContext.VO]

3、执行函数f1，创建f1的执行上下文，并入栈

ECStack = [f1Context,globalContext]

4、进入函数f1的执行上下文，创建f1.AO,同时函数f2也被定义,并将父作用域链保存到[[scope]]

f1Context = {
    AO:{
        arguments:{},
        scope:undefiend,
        f2:function(){}
    },
    scope:[AO,globalContext.VO],
    this:undefined
}

f2.[[scope]] = [f1Context.AO,globalContext.VO]（闭包根本）

5、f1函数开始执行，改变f1上下文的AO,同时将f2返回到外部(从而产生闭包)

f1Context = {
    AO:{
        arguments:{},
        scope:'local scope',
        f2:function(){}
    },
    scope:[AO,globalContext.VO],
    this:undefined
}

6、f1执行完成，退出上下文执行栈

ECStack = [globalContext]

7、函数f2接着执行，开始进入f2的执行栈

ECStack = [f2Context,globalContext]
f2Context = {
    AO:{
        arguments:{},
    },
    scope:[AO,f1Context.AO,globalContext.VO],
    this:undefined
}

8、f2执行,自己AO中没有找到，沿这作用域查找scope为local scope，最后退出上下文栈

ECStack = [globalContext]

# 闭包的作用

1、保存局部变量到内存中 2、让外部的函数可以访问函数内部的变量，相当于开辟了一个桥梁

# 闭包应用场景

1、解决块级作用域的问题

# 思考题

var stack = []
for(var i = 0;i<3;i++){
    stack[i] = function(){
        console.log(i)
    }
}
stack[0]()
stack[1]()
stack[2]()

上面的程序输出都是3 解析： 1、执行到stack[0]的时候，此时全局的上下文变量对象为glabloContext,同时也stack里面定义的函数也都保存了该变量对象到[[scope]]

glabloContext = {
    VO:{
        data:[...],
        i:3
    }
}
stack[0].[[scope]] = [glabloContext.VO]
stack[1].[[scope]] = [glabloContext.VO]
stack[2].[[scope]] = [glabloContext.VO]

2、开始进入stack[0]执行前，先创建执行上下文，这个时候作用域链变成了

stack[0]Context = {
    AO:{
        arguments:{}
    },
    scope: [AO,glabloContext.VO]
}

3、stack[0]开始执行的时候，发现AO没有找到i，就往VO中找到了i=3,data[1]和data[2]也是一样

如果需要输出0,1,2,下面我们修改成闭包来看看

var stack = []
for(var i = 0;i<3;i++){
    stack[i] = (function(i){
       return function(){
        console.log(i)
       }
    })(i)
}
stack[0]()
stack[1]()
stack[2]()

1、执行到stack[0]的时候，此时全局的VO为

glabloContext = {
    VO:{
        data:[...],
        i:3
    }
}

2、不过这里多了一步立即执行函数，创建了匿名函数的上下文，也被保存到stack[0]的stack的[[scope]]

匿名Context = {
   AO:{
    arguments:{
        0:0,
        length:1
    },
    i:0
   },
   scope:[glabloContext.VO]
}
stack[0].[[scope]] = [匿名Context.AO,glabloContext.VO]

3、进入stack[0]执行的前，创建上下文

stack[0]Context = {
    AO:{
        arguments:{}
    },
    scope:[AO,匿名Context.AO,glabloContext.VO]
}

4、执行stack[0],自己AO没有找到，就沿这链往上找的匿名AO,i=0,所以就输出0，data[1]和data[2]也是
