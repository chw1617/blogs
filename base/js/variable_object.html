# JS 变量对象

# 变量对象

  • 变量对象是执行上下文的重要属性之一
  • 变量对象是与执行上下文相关的数据作用域，存储了上下文定义的变量和函数
  • 下面我们主要分享全局作用域和函数作用域

# 全局变量对象VO

我们先理解一个全局对象

  • 全局对象是预定义的对象，比如js内部已经定义好的全局属性：NaN、undefined、infinity，还有全局函数：parseInt()、String()、decodeURLIComponent(),eval(),isNaN()等,这些变量和函数我们在浏览器里面都可以直接使用
  • 在顶层的javascript代码中，可以通过this引用全局对象
  • 全局对象其实就是全局的变量对象，全局对象是Object构造函数实例化的一个对象
  • 在浏览器中,全局对象有window属性指向本身
  • 下面我们可以伪代码写出全局对象的变量对象的原型
var num = 12
function f(){}

//variable object VO 可以表示为

VO = {
    ...内置属性
    num,
    f:referce to function(){}
}

# 函数变量对象AO

  • 在函数上下文中，我们使用活动对象 activity object 来表示
  • 其实活动对象和变量对象是一个东西，变量对象是内部引擎实现的，当一个函数执行的时候，创建一个对应的函数上下文，这个时候执行上下文的变量对象被激活，所以称为活动对象
  • 函数的执行上下文可以分为两个阶段：进入执行上下文、执行 1、函数进入执行上下文，这个时候代码还没执行，先创建活动对象，这个时候活动对象包含
  • 函数所有的形参和实参结合
  • 函数声明：指向对应声明的函数，有同名属性会覆盖（优先级最高，同全局上下文）
  • 变量声明：值为undefined,与声明的形参和函数同名不会干扰这些属性 下面是一个demo
function foo(a){
    var b = 'b'
    function c(){}
    var d = function(){}
    d = 3
}
foo(1)

先进入执行上下文，这个时候的活动对象AO为

AO =  {
    arguments:{
        0:1,
        length:1
    },
    a:1,
    b:undefined,
    c:function (){}
    d:undefined
}

foo(1)开始执行，修改赋值,这个时候AO

AO =  {
    arguments:{
        0:1,
        length:1
    },
    a:1,
    b:'b',
    c:function (){}
    d:3
}

上面的代码综合来总结可以整理为 1、 全局上下文的变量对象初始化为全局对象 2、 函数函数上下文变量对象初始化为arguments对象 3、 函数进去上下文时候，活动对象会添加会进行形参、变量、函数的初始化 4 、函数开始执行时候，修改活动对象的属性

# 思考题

 function print(){
     console.log(a) //Uncaught ReferenceError: a is not defined
     a = 1
 }
 print()

 解析：这个时候 a不是var声明的不存在提升也不在AO中，然后就从全局中找，全局中还没有就开始报错了

 function f2(){
     a = 2 //非严格模式是是隐式全局变量
     console.log(a) //2
 }
 f2()

 解析：这个时候a在AO中也拿不到，但是赋值给全局了，所以打印2

console.log(foo) // function foo(){}
var foo = 12
function foo(){}

解析：进入执行上下时候，会先处理函数声明，如果存在相同的属性会直接覆盖
